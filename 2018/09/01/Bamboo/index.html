Test2:接竹竿

Posted on 2018-09-01 | In 实战测试

Problem

问题描述

一天，神犇和 LCR 在玩扑克牌。他们玩的是一种叫做“接竹竿”的游戏。

游戏规则是：一共有 n 张牌，每张牌上有一个花色 c 和一个点数 v，花色不超过 k 种。将这些牌依次放入一列牌的末端。若放入之前这列牌中已有与这张牌花色相同的牌，你可以选择将这张牌和任意一张花色相同的牌之间的所有牌全部取出队列（包括这两张牌本身），并得到与取出的所有牌点数和相同的分数。现在已知 LCR 把这 n 张牌放入队列的顺序，求她最多能得多少分。

输入顺序即为 LCR 放入队列的顺序。即，ci表示第 i 张放入的牌的花色，vi表示第 i 张放入的牌的点数。

输入格式

第一行两个整数 n,k。

第二行，n 个整数 c1,c2,…,cn 表示花色，满足 1≤ci≤k。

第三行，n 个整数 v1,v2,…,vn 表示点数。

输出格式

输出一行一个整数，表示最多能得到的分数。

Analysis

定义dp[i]表示到第i张牌为止，可以得到的最大分数。

  • dp[i]=max(dp[i-1],dp[k]+sum[i]-sum[k-1]);
  • col[k]=col[i];
  • sum是前缀和;

因为n最大为10^6，所以直接枚举k肯定会超时，我优化了，但是可能力度不大，我是把上一个和颜色相同花色的位置记录到了next数组里，这样枚举到的都是和他颜色相同的牌，可以减少冗余枚举。但是如果花色过少，则会退化成枚举n次。

但是考试时没想到其他的，只好就先把这个程序打出来了，72分。

一开始由于next写成倒过来的，所以就把dp更新也倒过来了。

TLE Code

之后不太会优化了，就去请教了一下别人。

他告诉我这类题目一般要把二维改成一维，需要把后面的固定成某一常数。

有点单调队列优化的意味，不过我暂时还没有太理解。

  • dp[i]=max(dp[i-1],dp[k-1]+sum[i]-sum[k-1]);

k是和i颜色相同的点。可以看出，s对于所有的k，都统一加上了一个sum[i]，所以如果我们可以维护dp[k-1]-sum[k-1]的最大值，就可以直接更新。

可以用一个maxv数组，maxv[i]表示对于i颜色来说，到目前为止dp[k]-sum[k-1]的最大值。

注意，maxv一开始要赋为最小值，否则maxv为0，更新时会变成0+sum[i]，答案错误。
