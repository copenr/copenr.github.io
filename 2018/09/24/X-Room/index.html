Test3:小X的密室

Problem

小 X 正困在一个密室里，他希望尽快逃出密室。

　　密室中有 N 个房间，初始时，小 X 在 1 号房间，而出口在 N 号房间。

　　密室的每一个房间中可能有着一些钥匙和一些传送门，一个传送门会单向地创造一条从房间 X 到房间 Y 的通道。另外，想要通过某个传送门，就必须具备一些种类的钥匙（每种钥匙都要有才能通过）。幸运的是，钥匙在打开传送门的封印后，并不会消失。

　　然而，通过密室的传送门需要耗费大量的时间，因此，小 X 希望通过尽可能少的传送门到达出口，你能告诉小 X 这个数值吗？

　　另外，小 X 有可能不能逃出这个密室，如果是这样，请输出 “No Solution”。

Analysis

题目抽象为：从1开始走到n，每一条边都对于当前状态有一个钥匙的要求，在每一个点都可以得到一些钥匙。

而且每条边的权值都是1，直接BFS。

但是最多有10种钥匙，状态数过多，已开始没有想到二进制状压，发现前10个点都不需要考虑钥匙，于是就立马写了一个50分的BFS。

第二天，得知状压后，自己写了一下，思路其实差不多：

10位二进制，第i位为1表示当前已有第i种钥匙。

每次判断这条边能否经过，把当前已有的钥匙have和边需要的钥匙need进行或运算，得到的值如果和have相等，说明可以通过。

每次到一个点，只需要把已有的钥匙have和这个点有的钥匙key进行或运算，就可以得到当前的状态。

Wrong

一开始我的vhash只记录了点是否到过，是一维的。但是这道题和普通的BFS不一样的点就在于每个点还自带一个钥匙状态，也就是说同一个点如果钥匙状态不同，是可以重复入队的。否则就可能出现本身需要多绕路但有解的情况，被我的程序判成了No Solution。

Code O(n)

普通BFS就不放了，状压就是在它基础上改的。

开始时的95分：vhash只开一维记录点
