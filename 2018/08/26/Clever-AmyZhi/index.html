Test1:机智的AmyZhi

Posted on 2018-08-26 | In 实战测试

Problem

那年一个雨季，AmyZhi在校门外弯身买参考书。
这时SiriusRen走过来，一言不合甩给她一道（自认为）很难的题

给你一个数字N，求一个最小的正整数M，这个数字M的各个位的数字加上它本身之和恰好为N。

没有想到AmyZhi秒解了这道题 并把N扩大到了10^18 甩回给了SiriusRen

SiriusRen苦苦思索，发现并不会这道题
可怜SiriusRen一世英名，在AmyZhi面前却宛如智障少年
SiriusRen抬头透过倾斜的雨丝，看向模糊的世界，车水马龙，穿流不息，人来人往，形色匆匆，刹那间惘然，竟不知去向何方……

输入

第一行一个整数T，表示有T组数据。

接下来T行每行一个整数N。

输出

对于每组数据，输出M。

若无解，输出“Stupid SiriusRen”

样例输入

3

216

121

2005

样例输出

198

Stupid SiriusRen

1979

数据规模

对于20%的数据 N<=1000，T<=1000

对于40%的数据 N<=100000，T<=5

对于60%的数据 N<=100000，T<=5000

对于100%的数据 N<=1018，T<=50000

Analysis

看了一眼数据，T最大50000，也就是每一组数据处理不超过2000次最为保险。

然后我就开始了一条不归路，可能是因为被数学支配的恐惧，第一个做的事情就是打表，把1-10000所有的结果都打了出来，然而，这串数字的确有那么一些联系，但是规律我看了很久算了很久，还真没看出来。

然后想碰碰运气，看看枚举k-1000到k能不能出解，然后两个程序对拍了一下，发现居然答案全对，我经常是把答案写出来推过程，这次也不例外，我仔细想了一下，每一位数字最大是9，18位也就是162，也就是说答案不可能比k-162还要小，否则不可能是正解，结果我当时随便写了一个k-500，算复杂度忘记算拆分数字，结果TLE60了。

Code O(t*160*18)
