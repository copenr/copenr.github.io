输入格式

第1行: N (1 <= N <= 5000)，股票发行天数

第2行: N个数，是每天的股票价格。

输出格式

输出仅一行包含两个数:最大购买次数和拥有最大购买次数的方案数(<=231)当二种方案“看起来一样”时（就是说它们构成的价格队列一样的时候）,这2种方案被认为是相同的。

Analysis

读完题目就知道这是让求最长下降子序列，但是当我很块写完后却发现，这道题还让输出方案数，而且方案不能相同。

  • dp[i]=max(dp[i],dp[j]+1) a[j]>a[i]

然后我就在原来的程序上做了修改，当我们已经求出dp[i]后，我们只要枚举1~i-1所有的dp值，找到dp[j]=dp[i]-1 && a[j]>a[i]时说明i可以由j扩展来，如果不考虑方案相同可以直接全部累加，但是要考虑方案不同，我是这样想的：

如果多个相同数字满足条件，就加上这些数字中sum值最大的；如果只有一个数字满足条件，直接加上即可。

因为相同的数字一定是后面的优于前面的(选择更多)，所以从后往前枚举可以直接更新，一开始我用一个vhash记录每个数字有没有多个满足条件。

最后出解的时候想了很多方式都很麻烦，最后发现直接把a[n+1]定义为0，这样枚举时枚举到n+1，那么dp[n+1]一定是最长下降子序列+1，而sum[n+1]自然是最长下降子序列数目。

由于我的时间复杂度算的很粗略，考虑不周TLE了，每次要把vhash清零，有时还要枚举两次j，n最大为5000，完全有可能n^3/2*(memset复杂度)。

然后我束手无策，思考能不能不记录vhash或者避免每次清零。后来有这样一种方法：

满足dp[j]=dp[i]-1 && a[j]>a[i]条件的a[j]一定是从小到大排序的（包括相等），否则dp[i]一定不是最优。

所以无需记录vhash，只需要记录上一个a[j]的值，下一次比它小就行了。

Code O(n^2n)
